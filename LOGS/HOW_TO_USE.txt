BEST CASE (80% chance):

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "writefile not available" error
  Your executor doesn't support file operations.
  - Try a different executor (Synapse, KRNL recommended)
  - Output will still show in console (but you can't copy it)

PROBLEM: Script errors immediately
  - Make sure you loaded deobfuscate_dynamic.lua FIRST
  - Make sure you're in a Roblox game (not the menu)
  - Try a simpler script to test if executor works

PROBLEM: No files are created
  - Check your executor's workspace folder location
  - Make sure writefile permission is enabled in executor
  - Check executor console for errors

PROBLEM: Files are empty or incomplete
  - Wait longer - script might still be running
  - Files are auto-saved every 5 seconds
  - Type: _G.DeobfuscationFramework.printSummary() to force save

PROBLEM: No "deobfuscated_code.lua" file
  - Script doesn't use loadstring (this is normal for some scripts)
  - Check the other files (captured_strings.txt, api_calls.txt)
  - These still show what the script does

AFTER DEOBFUSCATION
================================================================================

Once you have the deobfuscated code:

1. READ IT CAREFULLY
   - Look for what it actually does
   - Check for malicious patterns
   - Verify it matches what you expected

2. LOOK FOR RED FLAGS
   - HTTP requests to unknown URLs
   - Accessing HttpService
   - Reading/sending player data
   - Accessing ServerStorage or ServerScriptService
   - Any Discord webhooks

3. MAKE YOUR DECISION
   - Safe to use? Great!
   - Suspicious? Don't use it
   - Unclear? Ask for help analyzing it
